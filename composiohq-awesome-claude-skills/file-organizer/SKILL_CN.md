# 文件组织器

这个技能充当您的个人组织助手，帮助您在计算机上维护干净、逻辑的文件结构，而无需不断手动组织的精神负担。

## 何时使用此技能

- 您的下载文件夹混乱不堪
- 您找不到文件，因为它们散落在各处
- 您有占用空间的重复文件
- 您的文件夹结构不再有意义
- 您想建立更好的组织习惯
- 您正在开始一个新项目，需要一个良好的结构
- 您在归档旧项目之前进行清理

## 此技能的功能

1. **分析当前结构**：审查您的文件夹和文件以了解您拥有什么
2. **查找重复项**：识别系统中的重复文件
3. **建议组织方式**：基于您的内容提出逻辑文件夹结构
4. **自动化清理**：在您的批准下移动、重命名和组织文件
5. **保持上下文**：基于文件类型、日期和内容做出明智的决策
6. **减少混乱**：识别您可能不再需要的旧文件

## 说明

当用户请求文件组织帮助时：

1. **了解范围**
   
   询问澄清问题：
   - 哪个目录需要组织？（下载、文档、整个主文件夹？）
   - 主要问题是什么？（找不到东西、重复项、太乱、没有结构？）
   - 有任何要避免的文件或文件夹吗？（当前项目、敏感数据？）
   - 组织的积极程度如何？（保守 vs. 全面清理）

2. **分析当前状态**
   
   审查目标目录：
   ```bash
   # 获取当前结构的概述
   ls -la [target_directory]
   
   # 检查文件类型和大小
   find [target_directory] -type f -exec file {} \; | head -20
   
   # 识别最大的文件
   du -sh [target_directory]/* | sort -rh | head -20
   
   # 计算文件类型
   find [target_directory] -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn
   ```
   
   总结发现：
   - 文件和文件夹总数
   - 文件类型细分
   - 大小分布
   - 日期范围
   - 明显的组织问题

3. **识别组织模式**
   
   基于文件，确定逻辑分组：
   
   **按类型**：
   - 文档（PDF、DOCX、TXT）
   - 图像（JPG、PNG、SVG）
   - 视频（MP4、MOV）
   - 归档（ZIP、TAR、DMG）
   - 代码/项目（包含代码的目录）
   - 电子表格（XLSX、CSV）
   - 演示文稿（PPTX、KEY）
   
   **按用途**：
   - 工作 vs. 个人
   - 活跃 vs. 归档
   - 项目特定
   - 参考资料
   - 临时/草稿文件
   
   **按日期**：
   - 当前年/月
   - 以前的年份
   - 非常旧（归档候选）

4. **查找重复项**
   
   当被请求时，搜索重复项：
   ```bash
   # 通过哈希查找完全重复项
   find [directory] -type f -exec md5 {} \; | sort | uniq -d
   
   # 查找同名文件
   find [directory] -type f -printf '%f\n' | sort | uniq -d
   
   # 查找大小相似的文件
   find [directory] -type f -printf '%s %p\n' | sort -n
   ```
   
   对于每组重复项：
   - 显示所有文件路径
   - 显示大小和修改日期
   - 建议保留哪个（通常是最新的或命名最好的）
   - **重要**：删除前始终请求确认

5. **提出组织计划**
   
   在进行更改之前呈现清晰的计划：
   
   ```markdown
   # [目录] 的组织计划
   
   ## 当前状态
   - X 个文件分布在 Y 个文件夹中
   - 总计 [大小]
   - 文件类型：[细分]
   - 问题：[列出问题]
   
   ## 提议的结构
   
   ```
   [目录]/
   ├── 工作/
   │   ├── 项目/
   │   ├── 文档/
   │   └── 归档/
   ├── 个人/
   │   ├── 照片/
   │   ├── 文档/
   │   └── 媒体/
   └── 下载/
       ├── 待分类/
       └── 归档/
   ```
   
   ## 我将进行的更改
   
   1. **创建新文件夹**：[列表]
   2. **移动文件**：
      - X 个 PDF → 工作/文档/
      - Y 个图像 → 个人/照片/
      - Z 个旧文件 → 归档/
   3. **重命名文件**：[任何重命名模式]
   4. **删除**：[重复项或垃圾文件]
   
   ## 需要您决定的文件
   
   - [列出任何您不确定的文件]
   
   准备好继续了吗？（是/否/修改）
   ```

6. **执行组织**
   
   获得批准后，系统地组织：
   
   ```bash
   # 创建文件夹结构
   mkdir -p "path/to/new/folders"
   
   # 移动文件并进行清晰的日志记录
   mv "old/path/file.pdf" "new/path/file.pdf"
   
   # 以一致的模式重命名文件
   # 示例："YYYY-MM-DD - 描述.ext"
   ```
   
   **重要规则**：
   - 删除任何内容前始终确认
   - 记录所有移动以便可能撤销
   - 保留原始修改日期
   - 优雅处理文件名冲突
   - 如果遇到意外情况，停止并询问

7. **提供摘要和维护提示**
   
   组织后：
   
   ```markdown
   # 组织完成！✨
   
   ## 发生了什么变化
   
   - 创建了 [X] 个新文件夹
   - 组织了 [Y] 个文件
   - 通过删除重复项释放了 [Z] GB
   - 归档了 [W] 个旧文件
   
   ## 新结构
   
   [显示新的文件夹树]
   
   ## 维护提示
   
   要保持组织：
   
   1. **每周**：整理新下载
   2. **每月**：审查并归档已完成的项目
   3. **每季度**：检查新的重复项
   4. **每年**：归档旧文件
   
   ## 为您准备的快速命令
   
   ```bash
   # 查找本周修改的文件
   find . -type f -mtime -7
   
   # 按类型整理下载
   [为您的设置定制命令]
   
   # 查找重复项
   [定制命令]
   ```
   
   想组织另一个文件夹吗？
   ```

## 专业提示

1. **从小处开始**：从一个混乱的文件夹（如下载）开始建立信任
2. **定期维护**：每周对下载进行清理
3. **一致命名**：对重要文件使用 "YYYY-MM-DD - 描述" 格式
4. **积极归档**：将旧项目移至归档而不是删除
5. **保持活跃分离**：在活跃和归档工作之间保持清晰的界限
6. **信任过程**：让 Claude 处理事情去向的认知负担

## 最佳实践

### 文件夹命名
- 使用清晰、描述性的名称
- 避免空格（使用连字符或下划线）
- 具体："client-proposals" 而不是 "docs"
- 使用前缀进行排序："01-current"、"02-archive"

### 文件命名
- 包含日期："2024-10-17-meeting-notes.md"
- 描述性："q3-financial-report.xlsx"
- 避免在名称中使用版本号（改用版本控制）
- 删除下载 artifacts："document-final-v2 (1).pdf" → "document.pdf"

### 何时归档
- 6 个月以上未触摸的项目
- 可能以后会参考的已完成工作
- 迁移到新系统后的旧版本
- 您犹豫删除的文件（先归档）
