# 定制化简历生成器

## 何时使用此技能

- 申请特定职位
- 为不同行业或角色定制简历
- 突出职业转型的相关经验
- 优化简历以通过 ATS（申请人跟踪系统）
- 为不同的工作申请创建多个简历版本
- 强调职位发布中提到的特定技能

## 此技能的功能

1. **分析职位描述**：从职位发布中提取关键要求、技能、资格和关键词
2. **确定优先级**：根据职位描述的语言和结构确定雇主最看重的内容
3. **定制内容**：重新组织并强调相关经验、技能和成就
4. **优化关键词**：在简历中自然融入 ATS 友好的关键词
5. **专业格式化**：创建适合各种格式的干净、专业的简历布局
6. **提供建议**：提出改进建议并突出需要解决的差距

## 如何使用

### 基本用法
提供职位描述和您的背景信息：

```
我正在申请这个职位：

[粘贴职位描述]

我的背景：
- 在 TechCorp 担任软件工程师 5 年
- 领导 3 名开发人员的移动应用项目团队
- 精通 Python、JavaScript、React
- 州立大学计算机科学学位
```

### 使用现有简历
上传或粘贴您当前的简历以及职位描述：

```
请为这个职位定制我的简历：

职位描述：
[粘贴职位描述]

我当前的简历：
[粘贴简历内容]
```

### 职业转型
当改变行业或角色时：

```
我正在从市场营销转型到产品管理。
这是我正在申请的工作：

[粘贴职位描述]

我的可转移经验：
- 7 年数字营销经验
- 领导跨职能团队
- 管理产品发布
- 数据分析和 A/B 测试
```

## 操作指南

当用户请求简历定制时：

### 1. 收集信息

**职位描述分析**：
- 如果未提供，请求完整的职位描述
- 询问公司名称和职位头衔

**候选人背景**：
- 如果用户提供现有简历，将其用作基础
- 如果没有，请求：
  - 工作经历（职位头衔、公司、日期、职责）
  - 教育背景
  - 关键技能和技术能力
  - 显著成就和指标
  - 认证或奖项
  - 任何其他相关信息

### 2. 分析职位要求

提取并优先考虑：
- **必须具备的资格**：经验年限、所需技能、教育
- **关键技能**：技术工具、方法、能力
- **软技能**：沟通、领导力、团队合作
- **行业知识**：特定领域经验
- **关键词**：重复的术语、短语和流行语，用于 ATS 优化
- **公司价值观**：职位描述中的文化契合度指标

创建心理映射：
- 优先级 1：关键要求（决定性因素）
- 优先级 2：重要资格（强烈期望）
- 优先级 3：理想技能（加分项）

### 3. 将候选人经验映射到要求

对于每个职位要求：
- 从候选人背景中识别匹配的经验
- 如果没有直接匹配，找到可转移的技能
- 注意需要解决或弱化的差距
- 识别要突出的独特优势

### 4. 构建定制化简历

**专业摘要**（3-4 行）：
- 以目标角色/领域的经验年限开头
- 包含职位描述中的前 3-4 项必需技能
- 如果相关，提及行业经验
- 强调独特的价值主张

**技术/核心技能部分**：
- 按与职位要求匹配的类别对技能进行分组
- 首先列出必需的工具和技术
- 使用职位描述中的准确术语
- 只包括您可以用经验证实的技能

**专业经验**：
- 对于每个角色，强调与职位要求一致的职责和成就
- 使用动作动词：领导、开发、实施、优化、管理、创建、分析
- **量化成就**：包括数字、百分比、时间范围、规模
- 重新排序要点以优先考虑最相关的经验
- 自然使用职位描述中的关键词
- 格式：**[动作动词] + [内容] + [如何/为什么] + [结果/影响]**

**教育**：
- 列出与职位相关的学位、认证
- 如果是早期职业生涯，包括相关课程
- 添加与职位要求匹配的认证

**可选部分**（如适用）：
- 认证和许可证
- 出版物或演讲
- 奖项和认可
- 志愿工作（如果与角色相关）
- 项目（特别是技术角色）

### 5. 为 ATS（申请人跟踪系统）优化

- 使用标准部分标题（专业经验、教育、技能）
- 自然融入职位描述中的准确关键词
- 避免表格、图形、页眉/页脚或复杂格式
- 使用标准字体和项目符号
- 同时包含缩写和完整术语（例如，"SQL (Structured Query Language)"）
- 在真实的情况下匹配职位头衔术语

### 6. 格式化和呈现

**格式选项**：
- **Markdown**：干净、可读、易于复制
- **纯文本**：ATS 优化，适用于所有系统
- **Word/PDF 提示**：提供格式化指导

**简历结构指南**：
- 经验 <10 年的保持 1 页，10+ 年的 2 页
- 使用一致的格式和间距
- 确保联系信息突出
- 使用倒序时间顺序（最近的优先）
- 保持干净、可扫描的布局，留有空白

### 7. 提供战略建议

在呈现定制化简历后，提供：

**优势分析**：
- 什么使该候选人具有竞争力
- 在求职信或面试中强调的独特资格

**差距分析**：
- 未完全满足的要求
- 解决差距的建议（课程、项目、重新构建经验）

**面试准备提示**：
- 与简历一致的关键谈话要点
- 基于职位要求准备的故事
- 展示契合度的问题

**求职信钩子**：
- 为求职信建议 2-3 个开场行
- 要扩展的关键成就

### 8. 迭代和完善

询问用户是否希望：
- 调整重点或语气
- 添加或删除部分
- 为不同角色生成替代版本
- 创建格式变体（传统 vs. 现代）
- 开发特定角色的版本（如果申请多个类似职位）

### 9. 应遵循的最佳实践

**应做**：
- 诚实准确 - 永远不要捏造经验
- 使用行业标准术语
- 用具体指标量化成就
- 为每个职位定制简历
- 校对语法和一致性
- 保持语言简洁有力

**不应做**：
- 包含个人信息（年龄、婚姻状况、照片，除非要求）
- 使用第一人称代词（I、me、my）
- 包含推荐信（"available upon request" 已过时）
- 如果职业生涯超过 20 年，列出每个工作（专注于相关、最近的经验）
- 使用未经定制的通用模板
- 超过 2 页，除非是非常高级的角色

### 10. 特殊考虑

**职业转换者**：
- 使用功能型或混合型简历格式
- 强调可转移技能
- 在摘要中创建引人注目的叙述
- 关注相关项目和课程

**应届毕业生**：
- 以教育开头
- 包括相关课程、项目、实习
- 强调学生组织中的领导力
- 如果 GPA 3.5+，包括 GPA

**高级管理人员**：
- 以执行摘要开头
- 关注领导力和战略影响
- 包括董事会成员资格、演讲
- 强调收入增长、团队建设、愿景

**技术角色**：
- 突出显示技术技能部分
- 列出编程语言、框架、工具
- 包括 GitHub、作品集或项目链接
- 提及方法（敏捷、Scrum 等）

**创意角色**：
- 包括作品集链接
- 突出创意成就和活动
- 提及工具和软件熟练度
- 考虑更具创意的格式（同时保持 ATS 兼容性）

## 获得最佳结果的提示

- **具体**：提供完整的职位描述和详细的背景信息
- **分享指标**：在描述您的经验时包括数字、百分比和可量化的成就
- **指明格式偏好**：让技能知道您是否需要 ATS 优化、创意或传统格式
- **提及约束**：分享任何特定要求（页面限制、要包含/排除的部分）
- **迭代**：不要犹豫要求修改或替代方法
- **多个申请**：为不同角色生成单独的定制版本

## 隐私说明

此技能处理您的个人和专业信息以生成定制化简历。在提交之前始终检查输出，确保准确性和适当性。删除或修改您不想与潜在雇主共享的任何信息。
